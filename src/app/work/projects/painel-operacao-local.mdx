## Visao geral

Este projeto nasce de uma dor recorrente em operacoes pequenas e medias: informacao espalhada demais para um time que precisa agir rapido.

Em vez de depender de planilhas, blocos manuais e leitura fragmentada, o painel concentra o que precisa ser visto para decidir e executar.

## Problema inicial

Quando lead, tarefa, status e proximo passo vivem em lugares diferentes, a operacao perde consistencia.

O custo disso nao e so organizacional. Ele aparece em atraso de resposta, retrabalho e leitura comercial fraca.

## Objetivos

- consolidar rotina comercial em uma unica interface
- reduzir etapas manuais do acompanhamento
- facilitar leitura de gargalos
- criar base util para automacoes futuras

## Como o sistema foi pensado

### Painel primeiro, detalhe depois

A estrutura prioriza uma leitura executiva da operacao antes de mergulhar em detalhes. Isso ajuda a tomar decisoes mais cedo e com menos atrito.

### Estados claros

Cada lead e tarefa precisa ter proximo passo evidente. O sistema evita excesso de status e trabalha com poucas etapas que realmente ajudam a mover a operacao.

### Automacao como extensao

O projeto nao usa automacao como efeito. Ela entra para reduzir tarefas repetitivas, lembrar follow-up e manter continuidade da execucao.

## Base tecnica

O sistema foi desenhado com Next.js e Supabase, com estrutura preparada para dashboards, filtros e historico operacional.

Essa base deixa o produto leve para uso diario e flexivel para evolucao conforme novas necessidades aparecem.

## Resultado esperado

- menos rotina manual
- visao comercial mais clara
- follow-up mais consistente
- base tecnica pronta para crescer

## Observacao

Este projeto e autoral. Ele existe para provar capacidade de pensar sistema, nao apenas pagina, e para mostrar como organizacao operacional tambem faz parte de uma entrega digital forte.
