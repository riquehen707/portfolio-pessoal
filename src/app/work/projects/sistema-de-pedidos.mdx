## Visao geral

Este projeto foi criado para resolver um problema comum em negocios locais: receber pedidos de forma mais organizada sem obrigar o cliente a baixar aplicativo.

A ideia foi construir um cardapio digital responsivo, rapido e com uma saida natural para o `WhatsApp`.

## Problema

Muitos cardapios digitais ficam pesados, confusos ou quebram no fluxo principal:

- navegar rapidamente pelos produtos
- personalizar pedido
- enxergar valor e adicionais
- finalizar sem atrito

## Solucao

Desenhei uma experiencia mobile first com:

- categorias claras
- produto com adicionais e observacoes
- resumo do pedido antes do envio
- mensagem final preparada para o `WhatsApp`

## Stack

Usei `Next.js` para performance e estrutura, `SCSS` para controle fino da interface e deploy em `Vercel` para manter o fluxo leve.

## Resultado

O sistema virou uma base reutilizavel para projetos de restaurantes e operacoes pequenas, principalmente quando o canal principal de venda e atendimento continua sendo o `WhatsApp`.

## Aprendizados

- mobile e o ambiente principal desse tipo de produto
- cada clique a menos aumenta a chance de pedido concluido
- a interface precisa servir a operacao, nao so ficar bonita
