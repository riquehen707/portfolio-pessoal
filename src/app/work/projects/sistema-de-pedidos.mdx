## Visão geral

Esse projeto nasceu para resolver um problema comum em negócios locais: receber pedidos de forma mais organizada sem obrigar o cliente a baixar aplicativo.

A ideia foi construir um cardápio digital responsivo, rápido e com uma saída natural para o `WhatsApp`.

## Problema

Muitos cardápios digitais ficam pesados, confusos ou quebram no fluxo principal:

- navegar rapidamente pelos produtos
- personalizar pedido
- enxergar valor e adicionais
- finalizar sem atrito

## Solução

Desenhei uma experiência mobile first com:

- categorias claras
- produto com adicionais e observações
- resumo do pedido antes do envio
- mensagem final preparada para o `WhatsApp`

## Stack

Usei `Next.js` para performance e estrutura, `SCSS` para controle fino da interface e deploy em `Vercel` para manter o fluxo leve.

## Resultado

O sistema virou uma base reutilizável para projetos de restaurantes e operações pequenas, principalmente quando o canal principal de venda e atendimento continua sendo o `WhatsApp`.

## Aprendizados

- mobile é o ambiente principal desse tipo de produto
- cada clique a menos aumenta a chance de pedido concluído
- a interface precisa servir a operação, não só ficar bonita
