## Visão geral

Este estudo de caso foi pensado para representar um tipo de trabalho que quero atrair: auditoria técnica para loja virtual que já existe, mas performa abaixo do potencial.

Aqui o foco não foi refazer tudo, e sim reorganizar estrutura, leitura de página e sinal técnico.

## Problema

Em projetos desse tipo, o padrão costuma ser:

- excesso de componentes sem hierarquia clara
- páginas de categoria fracas para busca
- produto com pouca escaneabilidade
- métricas sem leitura prática

## Abordagem

A proposta do case foi atacar quatro camadas:

1. arquitetura técnica
2. experiência de categoria
3. experiência de produto
4. mensuração de comportamento

## Stack

Usei `Vue` como base da experiência, `SCSS` para refinamento visual e `Google Analytics` como suporte para diagnóstico. O case reforça que meu trabalho com `SEO` não depende de framework específico.

## Resultado esperado

O ganho principal seria:

- páginas mais leves
- melhor leitura de oferta
- base mais forte para indexação
- menos atrito no caminho para compra
