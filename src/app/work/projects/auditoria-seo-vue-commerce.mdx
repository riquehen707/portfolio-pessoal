## Visao geral

Este estudo de caso foi pensado para representar um tipo de trabalho que quero atrair: auditoria tecnica para loja virtual que ja existe, mas performa abaixo do potencial.

Aqui o foco nao foi refazer tudo, e sim reorganizar estrutura, leitura de pagina e sinal tecnico.

## Problema

Em projetos desse tipo, o padrao costuma ser:

- excesso de componentes sem hierarquia clara
- paginas de categoria fracas para busca
- produto com pouca escaneabilidade
- metricas sem leitura pratica

## Abordagem

A proposta do case foi atacar quatro camadas:

1. arquitetura tecnica
2. experiencia de categoria
3. experiencia de produto
4. mensuracao de comportamento

## Stack

Usei `Vue` como base da experiencia, `SCSS` para refinamento visual e `Google Analytics` como suporte para diagnostico. O case reforca que meu trabalho com `SEO` nao depende de framework especifico.

## Resultado esperado

O ganho principal seria:

- paginas mais leves
- melhor leitura de oferta
- base mais forte para indexacao
- menos atrito no caminho para compra
