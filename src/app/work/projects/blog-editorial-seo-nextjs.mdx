## Visao geral

Este estudo de caso simula um blog com estrategia editorial forte, categorias bem organizadas e paginas preparadas para indexacao de longo prazo.

A proposta era construir um case mais alinhado ao meu trabalho com blogs, artigos profundos e estrutura tecnica para crescer organico.

## Problema

Muitos blogs publicam muito e estruturam pouco. O resultado costuma ser:

- categorias confusas
- paginas lentas
- artigos sem contexto interno
- baixa reaproveitabilidade de conteudo

## Solucao

O case foi pensado para mostrar um blog com:

- layout focado em leitura
- metadados consistentes
- taxonomia simples
- base pronta para interligacao editorial

## Stack

`Next.js` entrou pela performance, `SCSS` pelo controle visual fino e `Google Analytics` como camada de leitura de comportamento para evolucao futura.

## Resultado esperado

O estudo prova uma direcao clara para projetos editoriais:

- boa experiencia em mobile
- estrutura forte para SEO tecnico
- conteudo como ativo permanente, nao como pagina descartavel
